# Yoda GPT

A character-level language model trained on Yoda-style dialogue, built by following [Andrej Karpathy's miniGPT video](https://www.youtube.com/watch?v=kCc8FmEb1nY). The goal was to train a tiny GPT from scratch on a custom dataset and generate new Yoda-like sentences.

---

## Project Structure

```
yoda-gpt/
├── bigram.py           # Baseline bigram model (no attention)
├── gpt.py              # Full transformer-based GPT
├── yoda_dialogue.txt   # Training dataset (~1.1MB of Yoda dialogue)
└── yoda_generated.txt  # Sample output from the trained model
```

---

## How It Works

### Step 1 — Bigram baseline (`bigram.py`)
A bigram model has no memory or attention. It learns one thing: given character X, what character is most likely to follow? It's the simplest possible language model and serves as a sanity check that the data pipeline works.

**Result:** Val loss ~2.35. Output is mostly noise but `YODA:` and common letter patterns start appearing.

### Step 2 — Transformer GPT (`gpt.py`)
The full model adds:
- **Token + positional embeddings** — each character gets a learned vector, plus a sense of where it sits in the sequence
- **Multi-head causal self-attention** — each token can look back at previous tokens and decide what's relevant
- **Feed-forward layers** — per-position computation after attention
- **Residual connections + LayerNorm** — training stability

**Result:** Val loss dropped from ~4.0 → ~1.65 in one run, and the model learned Yoda's inverted syntax structure.

## Results

### Bigram output
```
YODA: mo llldald, se pIs, SInis ong, wnochacelorour t tites...
```
Mostly random characters. Knows `YODA:` exists but nothing else.

### GPT output (2000 steps, small config)
```
YODA: Patient all y you must. The fix asult bout, refectives they are.
YODA: Proctrictions recant intnally. Ond than engineeciation, only.
```
Real words starting to appear. Yoda's inverted grammar (`X, you must`) is clearly learned even though individual words are still garbled.

### GPT output (5000 steps, larger config — `yoda_generated.txt`)
```
YODA: Anticipate traffic flow, coast tops rather than brake. Momentum conserved, brakes last longer.
YODA: Two-factor authentication, enable it everywhere at be.
YODA: Rest meat at after cooking, always.
```
Some lines are nearly perfect. Others are a mix of real words in the right structure but with corrupted spelling mid-sentence.

---

## Why the Output Wasn't Perfect

### 1. The dataset had repeated lines
The initial dataset was generated by writing ~300 lines in Python and repeating them until the file hit 1MB. The model quickly memorized these lines rather than learning generalizable patterns. This explains why val loss dropped suspiciously low (~0.32) in one run — the model was reciting training data, not generating.

**Fix:** Regenerate the dataset by calling a language model API once per topic, getting 25 fresh lines each time, for ~100 topics. Every line unique.

### 2. Training on CPU limits model size
Running on an Intel Mac CPU means the full Karpathy hyperparameters (`block_size=256, n_embd=384, n_layer=6`) either time out or take hours. The configs used here were scaled down significantly:

| Parameter | Karpathy original | This project |
|-----------|------------------|--------------|
| `block_size` | 256 | 64–128 |
| `n_embd` | 384 | 256 |
| `n_layer` | 6 | 4 |
| `n_head` | 6 | 4 |

Smaller `block_size` means the model only sees ~64–128 characters of context at a time. Yoda's sentences are often longer than that, so it loses the thread mid-sentence.

**Fix:** Run on a machine with a GPU. Even a free Google Colab T4 GPU would run the full config in ~15 minutes.

### 3. Character-level modeling is hard
This model predicts one character at a time. To spell the word "authentication" correctly it must make 14 correct predictions in a row. Word-level or subword tokenization (like BPE, used in real GPT models) would make this much easier — the model would predict whole words or word-pieces instead of individual letters.

---

## What It Did Demonstrate

Despite the limitations, the project successfully shows:

- **The transformer architecture works** — attention heads, residual connections, LayerNorm all training correctly from scratch
- **Syntax is learnable** — the model picked up Yoda's object-first inverted grammar (`X, you must. Y, it does.`) without being explicitly told the rule
- **The `YODA:` format is learned** — the model consistently produces properly formatted output lines
- **Loss curves make sense** — watching train loss drop from 4.0 → 1.0 while val loss tracks it (or diverges when overfitting) is exactly what the theory predicts
- **Bigram vs GPT gap is real** — val loss of 2.35 (bigram) vs 1.65 (GPT) is a meaningful improvement, and it's visible in the output quality

## How to Run

```bash
# Install dependencies
pip install torch

# Run the bigram baseline first (~2 min on CPU)
python bigram.py

# Run the full GPT (~15–20 min on CPU with scaled-down config)
python gpt.py
```

Both scripts expect `yoda_dialogue.txt` in the same directory.

---

## Hyperparameters Used (CPU-friendly)

```python
batch_size    = 48
block_size    = 128
max_iters     = 5000
n_embd        = 256
n_head        = 4
n_layer       = 4
dropout       = 0.2
learning_rate = 3e-4
```

For GPU, restore the full Karpathy settings: `block_size=256, n_embd=384, n_head=6, n_layer=6`.
